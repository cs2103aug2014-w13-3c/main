The following is a list of Effi command keywords and option fields.

Keywords
/v (view type)view (view type): switches between agenda view, weekly view and monthly view
/a (task name) [-<option> <value> … ] or add (task name) [-<option> <value> … ]: add a task with an optional start and end time.
/u (task name) [-<option> <value> … ] or update (task name) [-<option> <value> … ]: update a task.
/d (task name) [-<option> <value> … ] or delete (task name) [-<option> <value> … ]: deletes a task with the specified details. If there are tasks with the same name and no timing is specified, the delete will fail and the user will be prompted to enter the timings to distinguish them.
/c (task name) or complete (task name): marks the task as complete.
/s (boolean function) or search (boolean function): search for items. See below for syntax.
/f (boolean function) or filter (boolean function): Filter is a more permanent search - this modifies what is available to all other components. The syntax is similar to that of searching.
/z or undo: undo the most recent action.
/? or help: produces the list of available commands.
exit: exits the program.

Instead of specifying an existing issue’s name, you can simply specify the task ID via #ID.

Options

Where applicable, [-<option> <value> … ] refers to a list of options and values in any order.

The available options are:
-s or start
-e or end
-p or priority
-r or repeat
-t or tag
-n or rename renames the task, and is only applicable to Update.
